# DNS 管理器 (Dlist) - 项目开发指南
# 适用于 OpenCode / AI 角色团队执行

=========================================
一、项目说明
本项目为 Flutter 跨平台多服务商域名 & DNS 集中管理工具
采用插件化驱动架构，支持 Cloudflare、DNSHE 等多家服务商
支持 MD3 / Cupertino 双UI风格、全局深浅主题、多语言、安全加密存储、操作审计、应用内支付订阅

=========================================
二、目录结构（必读）
1. API 文档目录
   路径：./api-docs
   内容：Cloudflare、DNSHE 等服务商官方开放 API 文档
   用途：所有开发、建模、对接的唯一依据

2. 通用规范目录
   路径：./rules
   内容：项目通用开发规则、角色职责、技术规范、架构约束
   用途：AI 必须全程遵守，不可偏离

=========================================
三、核心工作流程（强制执行）
1. 先读取 ./rules 内全部通用规范
2. 读取 ./api-docs 内全部服务商 API 文档
3. 按标准流程开发：API整理 → 架构 → 模型 → 驱动 → UI → 安全 → 测试
4. 任何代码或文档改动前，必须先将现有代码成功 push 到 GitHub
5. 全部功能开发并测试通过后，执行最终构建打包

=========================================
四、必须输出的构建产物
1. Android 平台
   - arm64-v8a Release APK
   - x86_64 Release APK
2. Windows 平台
   - Windows 桌面端 x64 Release 程序

=========================================
五、基础约束（必须遵守）
1. 未 push → 不开发
2. 未测试 → 不打包
3. 所有开发必须以 api-docs 文档为唯一依据
4. 全程严格遵守 rules 目录内的所有规范
5. 支付订阅模块必须安全防漏洞、防篡改、不本地验权
6. MD3 与 Cupertino 双风格纯净化渲染，不混用控件
7. 服务商驱动完全隔离，列表 UI 由驱动自治实现
8. 用户凭证加密存储，关键操作记录审计日志
