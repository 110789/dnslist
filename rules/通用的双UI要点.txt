# Flutter 双风格(MD3 / Cupertino)自定义切换 开发规范要点
# 适用：交给OpenCode严格遵守，全覆盖避坑约束

一、整体架构规则
1. 分为两个独立配置维度：UI风格、深浅明暗模式
2. 深浅模式全局共用【同一个开关、同一个持久化标记】，不拆分两套控制
3. UI风格独立配置：MD3 / Cupertino 二选一
4. 本地持久化只存两个Key：
   - ui_style：记录当前是MD3还是Cupertino
   - app_dark_mode：记录浅色/深色/跟随系统，全局共用不随风格重置
5. 首次启动默认规则：
   - 安卓设备默认：UI风格=MD3，深浅=跟随系统
   - iOS设备默认：UI风格=Cupertino，深浅=跟随系统
6. 切换UI风格时，保留用户已选的深浅模式状态，仅切换对应风格的亮暗色板，不重置用户明暗偏好

二、配色与Material You动态取色规则
1. 内部维护4套独立色板，互不串色：
   - MD3 亮色 / MD3 暗色
   - Cupertino 亮色 / Cupertino 暗色
2. 动态取色(Material You)仅允许生效条件：
   - 必须同时满足：安卓系统 + 当前UI风格为MD3
3. 禁用动态取色强制规则（满足任一即关闭壁纸取色）：
   - iOS系统
   - 安卓但用户手动切换到Cupertino风格
4. Cupertino风格无论运行在安卓还是iOS，永远禁用Material You，强制使用预设固定色板
5. 深浅模式切换逻辑全局统一，切换UI风格自动适配该风格的亮暗色配色

三、控件与弹窗统一适配规则
1. 禁止业务页面到处手写if/else判断风格或平台
2. 所有原生控件必须封装自适应组件，底层统一判断，业务层直接调用：
   - 对话框：AlertDialog / CupertinoAlertDialog 自适应封装
   - 底部弹窗：ModalBottomSheet / CupertinoActionSheet 自适应封装
   - 日期时间选择器：MD3日历样式 / Cupertino滚轮样式 自适应
   - 下拉菜单、分段控制器、标签栏、开关、复选框、单选框、滑块
   - 输入框：TextField / CupertinoTextField 自适应
3. 全局视觉纯净约束：
   - Cupertino模式下不得露出任何MD3原生控件
   - MD3模式下不得露出任何Cupertino原生控件

四、导航、路由与动画规则
1. 根App不写死MaterialApp/CupertinoApp，根据用户配置动态渲染根组件
2. 页面转场动画跟随当前UI风格自动切换：
   - MD3：安卓标准推入动画
   - Cupertino：iOS右侧划入+弹性返回动画
3. 导航栏封装自适应：AppBar / CupertinoNavigationBar 统一封装
4. 返回按钮图标自动适配MD3箭头 / Cupertino苹果风格箭头
5. Cupertino模式必须启用iOS右滑返回手势，MD3遵循安卓标准逻辑

五、系统UI与沉浸式适配
1. 状态栏文字亮度、沉浸式模式，需分别适配：亮/暗模式 + MD3/Cupertino组合场景
2. 安卓底部虚拟导航栏配色，Cupertino模式不能沿用MD3底色，需匹配苹果风格配色
3. 滚动阻尼效果区分：
   - Cupertino启用苹果弹性阻尼
   - MD3使用安卓常规滚动效果

六、字体、间距、圆角设计规范
1. 两套风格独立排版规范：字号、行高、字体基线分开定义
2. 圆角、内边距、分割线规范全局统一，按风格区分，禁止各页面随意写死尺寸
3. Cupertino与MD3各自遵循自身设计规范，不混用圆角、间距样式

七、渲染与兼容约束
1. 开启Impeller渲染后，需保证Cupertino磨砂、阴影、渐变无黑块、无模糊失效
2. Cupertino风格编译到Android APK需正常渲染，无布局错位、溢出、比例异常
3. 适配平板、折叠屏布局，MD3与Cupertino各自遵循对应平台大屏设计规范

八、持久化与设置逻辑
1. UI风格、深浅模式配置永久本地持久化，重启、杀进程、版本更新不重置
2. 设置页面可将风格切换、深浅开关放在同一面板展示，底层仍为两个独立存储字段
3. 仅用户手动修改时变更配置，切换风格不主动篡改深浅模式偏好

九、禁止开发行为
1. 禁止将深浅模式与UI风格共用同一个存储标记
2. 禁止拆分两套深浅模式开关，必须全局唯一控制
3. 禁止在业务逻辑中散落平台/风格判断，所有自适应逻辑收敛到底层封装组件
4. 禁止Cupertino风格强行套用安卓Material You动态取色
